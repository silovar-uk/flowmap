# Flowmap

図を直接操作しながら、業務の流れ・依存関係・補足情報を整理するローカルWebアプリです。

## v0.4

- キャンバスを中心にしたGUI
- 初回表示でイベント業務のサンプルを展開
- ノードをドラッグして配置
- ノード右端の接続点からドラッグして依存関係を作成
- 矢印を選択して接続元・接続先を変更
- フェーズ、作業グループ、TODOを図上で整理
- 期限、担当、場所、関連リンク、補足を右側のインスペクターで編集
- 細かな作業はチェック項目としてノード内に保持
- グループの移動・折りたたみ・自動整列
- ズーム、パン、全体表示、ミニマップ
- Undo / Redo
- JSON / YAMLの入出力
- 全体図とTODO一覧のPDF出力
- 純粋なベクターSVGによるfavicon・ブランドアイコン
- AI・ログイン・外部API不要

## 基本操作

- キャンバスの空白をダブルクリック：作業を追加
- ノードをドラッグ：作業を移動
- ノード右端の点からドラッグ：別の作業へ接続
- 矢印をクリック：関係性を選択
- グループ見出しをドラッグ：グループと内部作業を移動
- グループの「−／＋」：折りたたみ・展開
- マウスホイール／上部ボタン：拡大・縮小
- `Ctrl / Cmd + Z`：元に戻す
- Delete：選択した作業または関係を削除

## 公開

ビルド工程を必要としない静的サイトのため、GitHub Actionsではなく、`main`ブランチのルートから直接GitHub Pagesへ公開します。

初回のみ、リポジトリで次を設定してください。

1. **Settings → Pages** を開く
2. **Build and deployment → Source** を **Deploy from a branch** にする
3. Branchを **main**、Folderを **/ (root)** にする
4. **Save** を押す

公開URL：

`https://silovar-uk.github.io/flowmap/`

反映には数分かかることがあります。リポジトリには `.nojekyll` を置き、ファイルを加工せずそのまま配信します。

## ローカルで開く

```bash
python -m http.server 8000
```

その後、`http://localhost:8000` を開きます。

## データについて

データはブラウザのlocalStorageへ保存されます。v0.4では旧版と保存領域を分け、空の旧データを引き継がない構成です。別端末との同期は行いません。必要に応じてJSONまたはYAMLを書き出してください。

## 設計原則

- 図を基本にする
- ノード、矢印、囲みを直接操作する
- 補足は選択した項目へ後から加える
- 親子関係は囲み、依存関係は矢印で表す
- すべてを細かなノードへ分解しない
- チェック項目はノード内へ吸収する
- 独立したTODOを無理につながない
- 装飾だけでなく、関係性の見分けやすさと操作速度を優先する

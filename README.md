# Flowmap

付箋と囲みを直接操作しながら、業務の流れ・依存関係・補足情報を整理するローカルWebアプリです。

## v0.5

- 明るいホワイトボードを中心にしたGUI
- TODOを付箋、作業グループを囲みとして表示
- 付箋をドラッグして配置
- 付箋右端の接続点からドラッグして依存関係を作成
- 矢印を選択して接続元・接続先を変更
- フェーズ、囲み、付箋を図上で整理
- 期限、担当、場所、関連リンク、補足を右側の補足メモで編集
- 細かな作業はチェック項目として付箋内に保持
- 囲みの移動・折りたたみ・自動整列
- ズーム、パン、全体表示、ミニマップ
- Undo / Redo
- JSON / YAMLの入出力
- 全体構造と付箋一覧のPDF出力
- AI・ログイン・外部API不要

見た目はホワイトボードと付箋に寄せていますが、手描き風の曖昧さは入れず、接続・整列・期限・検索などの実務機能を優先しています。

## 基本操作

- キャンバスの空白をダブルクリック：付箋を追加
- 付箋をドラッグ：作業を移動
- 付箋右端の点からドラッグ：別の付箋へ接続
- 矢印をクリック：前後関係を選択
- 囲みの見出しをドラッグ：囲みと内部の付箋を移動
- 囲みの「−／＋」：折りたたみ・展開
- マウスホイール／上部ボタン：拡大・縮小
- `Ctrl / Cmd + Z`：元に戻す
- Delete：選択した付箋または関係を削除

## 公開

ビルド工程を必要としない静的サイトのため、GitHub Actionsではなく、`main`ブランチのルートから直接GitHub Pagesへ公開します。

初回のみ、リポジトリで次を設定してください。

1. **Settings → Pages** を開く
2. **Build and deployment → Source** を **Deploy from a branch** にする
3. Branchを **main**、Folderを **/ (root)** にする
4. **Save** を押す

公開URL：

`https://silovar-uk.github.io/flowmap/`

反映には数分かかることがあります。リポジトリには `.nojekyll` を置き、ファイルを加工せずそのまま配信します。

## ローカルで開く

```bash
python -m http.server 8000
```

その後、`http://localhost:8000` を開きます。

## データについて

データはブラウザのlocalStorageへ保存されます。v0.4以降は旧版と保存領域を分け、空の旧データを引き継がない構成です。別端末との同期は行いません。必要に応じてJSONまたはYAMLを書き出してください。

## 設計原則

- 図を基本にする
- 付箋、矢印、囲みを直接操作する
- 補足は選択した項目へ後から加える
- 親子関係は囲み、依存関係は矢印で表す
- すべてを細かな付箋へ分解しない
- チェック項目は付箋内へ吸収する
- 独立したTODOを無理につながない
- 見た目は親しみやすく、操作と情報密度は実務向けに保つ

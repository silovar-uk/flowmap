# Flowmap

図形と囲みを直接操作しながら、業務の流れ、前後関係、補足情報を整理するローカルWebアプリです。

## v0.13.0

フローを一目で追いやすくしながら、各図形にはより多くの補足情報を蓄えられるようにしました。

- 矢印の中間にも進行方向を表示
- 選択した矢印の接続元を「元」、接続先を「先」と表示
- 通常の処理カードを224×116pxから184×88pxへ縮小
- 付箋上はタイトル、状態、タグ2件、メモ件数、チェック進捗へ整理
- タグを1件ずつ追加・削除でき、最大30件まで保存
- 概要メモを最大12,000文字まで保存
- 論点別の追加メモを最大30件まで作成
- 検索対象に短い補足、概要メモ、追加メモを追加
- `Tab`で作る次の処理は、元の図形から矢印で接続

## 矢印の方向

矢印は、終端の矢尻だけでなく線の中間にも小さな方向記号を表示します。

矢印を選択すると、次の表示を追加します。

- 接続元の図形を暖色の枠で強調し「元」と表示
- 接続先の図形を青い枠で強調し「先」と表示
- 線の始点と終点にも「元」「先」のバッジを表示
- 右パネルに`接続元 → 接続先`を表示

図形を近づけて接続する時の方向プレビューと、上下左右の接続点からドラッグする操作も残しています。

## コンパクトな付箋

基本サイズは次の通りです。

```text
処理・入出力   184 × 88px
開始／終了     184 × 72px
判断           190 × 104px
書類           184 × 92px
```

付箋上には、流れを読むために必要な情報だけを表示します。

- 図形の種類
- タイトル
- 状態
- 期限の月日
- タグ2件と残り件数
- メモ件数
- チェックリスト進捗

期限、担当、場所、リンク、長文メモなどは右パネルへ集約します。

## 補足情報

### 短い補足

基本情報タブに、240文字までの短い補足を保存できます。検索対象にも含まれます。

### タグ

タグはカンマ区切りの一括入力ではなく、チップ形式で管理します。

- 入力して`Enter`、`,`、`、`で追加
- タグ横の`×`で削除
- 同じタグの重複を防止
- 1図形につき最大30件
- 付箋上には先頭2件と残り件数だけ表示

### 概要メモ

メモタブの概要メモには、背景、注意事項、判断理由などを最大12,000文字まで保存できます。

### 追加メモ

概要メモとは別に、論点ごとのメモを複数作成できます。

各追加メモは次の項目を持ちます。

- 見出し：最大120文字
- 本文：最大5,000文字
- 更新日時

1図形につき最大30件です。

## ヘッダーとボード管理

- 現在のボード名をヘッダーに常時表示
- ボード名を押すと、保存一覧、新規作成、名前変更、複製、削除を表示
- 編集内容はIndexedDBへ自動保存
- 保存状態は「保存中…／保存済み／保存失敗」で表示
- PDF、JSON、YAML、読み込み、全体削除は**書き出し**へ集約
- ボード名メニューは`Esc`で閉じる

## キャンバス上の追加ツール

- **＋処理**：既定の処理を追加
- 右隣の矢印：開始／終了、処理、判断、入出力、書類を選択
- **囲み**：仕事のまとまりを追加
- **フェーズ**：大きな区切りを追加
- **整える**：現在の囲み、現在のフェーズ、ボード全体から範囲を選択

左の構造一覧と右の詳細パネルは、最初は閉じた状態で開始します。右パネルは対象を選択した時だけ開きます。

## 空白クリックと仮図形

- 空白をクリックすると、その場所に薄い仮図形を表示
- 名前を入力して`Enter`を押した時だけ確定
- 未入力のまま別の場所を押すと破棄
- `Esc`でも破棄
- `Shift + Enter`はタイトル内の改行
- マウスで8px以上、タッチで12px以上動かすとボード移動

## タイトル編集とキーボード

- `F2`：選択した図形の名前を編集
- `Enter`：タイトル編集を開始／内容を確定
- `Shift + Enter`：タイトル内で改行
- `Tab`：下に次の処理を追加し、元の図形から矢印で接続
- `Ctrl / Cmd + Enter`：右に子の処理を追加して接続
- `Ctrl / Cmd + D`：複製
- `Delete`：削除
- `Space + ドラッグ`：図形の上からボードを移動
- `F`：選択項目または全体へズーム
- `Ctrl / Cmd + K`：検索
- `Ctrl / Cmd + Z`：元に戻す

## 使い方レクチャー

レクチャーは自動では表示しません。ヘッダーの**使い方**を押した時だけ、一時的な練習ボードで始まります。

練習中の追加、移動、接続はIndexedDBへ保存されません。終了すると、開始前のボードと選択状態へ戻ります。

## 保存方式

IndexedDBデータベース`flowmap`を使用します。

```text
flowmap
├─ boards
│  └─ board_xxx
│     ├─ name
│     ├─ state
│     ├─ createdAt
│     ├─ updatedAt
│     └─ schemaVersion
└─ meta
   ├─ activeBoardId
   └─ uiV12Prepared
```

旧データは必要に応じてIndexedDBへ自動移行します。新しい補足項目が存在しない旧図形には、空の短い補足と追加メモ配列を自動で補います。

## 全体を削除

**書き出し → 全体を削除**から、現在のボードにある図形、矢印、囲み、フェーズをすべて削除できます。

- ボード名と保存枠は残る
- 他の保存済みボードには影響しない
- 確認画面を経由する
- 実行後は元に戻せない

## フローチャート記法

- **開始／終了**：角丸の端子形
- **処理**：長方形
- **判断**：ひし形
- **入出力**：平行四辺形
- **書類**：下辺が波形の書類形
- **矢印**：上下左右から直角接続
- **分岐ラベル**：判断から出る矢印に「はい／いいえ」などを表示
